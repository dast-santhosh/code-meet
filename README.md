# DevShaala - Code & Meet Workspace 🚀

A premium virtual classroom app combining real-time multi-peer video meetings and collaborative Python programming. Designed for **DevShaala**, it features a VS Code-inspired sidebar, real-time code editor synced via **Yjs**, client-side Python execution via **Pyodide WASM**, and multi-peer video feeds via **WebRTC**.

---

## 🏗️ Project Architecture & Stack

### Frontend: React (Vite) + Monaco Editor + Yjs
- **Collaborative Editor:** Monaco Editor bound with Yjs CRDTs. Synchronizes multi-user keystrokes and cursors instantly.
- **Python WASM Execution:** Python executes fully client-side inside a browser sandbox using **Pyodide**, enabling imports for `numpy`, `pandas`, and matplotlib rendering output maps onto a canvas.
- **Draggable Overlay Portals:** Chat, AI Chat, and DuckDuckGo search operate inside movable frames using Framer Motion.
- **Firebase client:** Direct authentication and squadron checks using Firebase Firestore.

### Backend: Python FastAPI WebSockets Relay
- **Stateless Signaling:** Relays RTC packet signaling (offers, answers, ICE candidates) in a peer-to-peer mesh configuration.
- **In-Memory Cache:** Remembers session chat history and Yjs document updates so late-joining students sync instantly.

---

## 🌟 Premium Meet & IDE Features

### 💻 Commandant Collaborative Editing
- **Instructor Authority:** The Commandant can click the **Edit Code** icon next to any Cadet's name in the People panel or Right Previews. The center Monaco editor will load that Cadet's workspace in write-mode, enabling direct remote debugging.
- **Cadet View Limitations:** Cadets can only view other peers' codes in read-only previews. They can never modify others' code workspaces.
- **Editing Indicator:** When editing someone else, a clear banner is rendered at the top of the editor with a "Reset to Self" button to easily return.

### 📱 Mobile UI Layout (Auto-Focus Mode)
- **Compact Sidebar:** Hides heavy panels (Console, Previews) and presents a streamlined activity bar.
- **Active Coding State:** When a Cadet focuses on the editor (opening the soft keyboard), the editor expands to take up 100% height to ensure typing accessibility.
- **Standby View:** When the editor is blurred, the Commandant's video feed expands automatically in a large bottom panel.

### 🎙️ Audio Control & Permissions (Cadet Mute Locks)
- **Locked Mic by Default:** Cadets are joined in a muted state and their mic toggle is locked.
- **Speak Request Modal:** Clicking the mute button sends a permission alert. The Commandant receives a centered popup modal to either **Grant** or **Deny** microphone activation.
- **Private Audio Whispering:** The Commandant can click the **Mic icon** next to a Cadet in the People list. This initiates a secure peer whisper channel where the Commandant's voice is only routed to that cadet, automatically muting the Commandant's audio track for all other participants.

## 🚀 Local Setup & Operation

### 1. Run the FastAPI Backend
1. Navigate to the backend folder:
   ```bash
   cd backend
   ```
2. Create and activate a virtual environment:
   ```bash
   python -m venv venv
   venv\Scripts\activate
   ```
3. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```
4. Boot the server:
   ```bash
   python run.py
   ```
   *The backend will boot on `http://localhost:8000`.*

### 2. Run the React Frontend
1. Navigate to the frontend folder:
   ```bash
   cd frontend
   ```
2. Install dependencies:
   ```bash
   npm install
   ```
3. Run the Vite development server:
   ```bash
   npm run dev
   ```
   *The client will boot on `http://localhost:5174`.*

## 🌐 Cloud Deployment Walkthrough (Vercel + Render)

Because **Vercel** operates serverless functions, it does not support long-running, stateful WebSocket connections. Therefore, the FastAPI WebSocket backend must be hosted on a persistent container provider like **Render**.

### Step A: Deploy the Backend on Render (Render Blueprint)
1. The repository contains a `render.yaml` blueprint in the root directory.
2. In your Render Dashboard, click **New +** -> **Blueprint**.
3. Connect your GitHub repository. Render will automatically read `render.yaml`, configure the `backend/` directory, set the build commands, and provision the server with `$PORT`.
4. *(Fallback Manual Setup)*: Create a **Web Service**, set the Root Directory to `backend`, Build Command to `pip install -r requirements.txt`, and Start Command to `python run.py`. Add environment variable `PORT=8000`.
5. Copy the primary URL generated by Render, e.g., `https://devshaala-codemeet-backend.onrender.com`.

### Step B: Deploy the Frontend on Vercel
1. Set up your React application project on Vercel. Connect your repository.
2. Set the Root Directory to `frontend`.
3. In the Vercel project settings, navigate to **Environment Variables** and define:
   - **Key:** `VITE_WS_URL`
   - **Value:** The secure WebSocket URL pointing to your Render backend (replace `https://` with `wss://`):
     - `wss://devshaala-codemeet-backend.onrender.com`
   - **Key:** `VITE_GROQ_API_KEY`
   - **Value:** Your Groq Cloud API Key (for the draggable AI Tutor chat widget).
4. Click **Deploy**. The React client will build and securely connect to the hosted WebSockets relay server for real-time video/audio signaling and Yjs sync!
